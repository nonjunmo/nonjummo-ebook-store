# 논준모연구소 e-book 판매 사이트 설계

## 목표

논준모연구소의 e-book 교재를 판매하는 운영형 웹사이트를 만든다. 사용자는 상품 목록에서 장바구니에 담거나 바로 주문할 수 있고, 주문 페이지에서 주문 내역과 입금 안내를 확인한 뒤 개인정보를 입력해 주문을 제출한다. 관리자는 상품을 등록하고 주문별 입금확인 및 발송완료 상태를 관리한다.

## 기술 방향

Render Web Service와 SQLite를 기준으로 구현한다. 서버는 Node.js, Express, EJS를 사용해 서버 렌더링 방식으로 구성한다. SQLite 파일 경로는 `DATABASE_PATH` 환경변수를 사용하며, Render 운영 환경에서는 Persistent Disk의 `/var/data/app.db`를 사용한다.

## 화면

메인 화면은 첨부 이미지의 구조를 따른다. 상단에는 로고, `e-book 목록 보기`, `관리자 로그인` 링크를 둔다. 본문에는 안내 문구와 e-book 목록을 10개씩 보여준다. 각 상품은 표지, 제목, 저자/소속, YouTube 멤버십 표시, 간략소개/목차보기, 가격, 장바구니 버튼, 주문하기 버튼을 가진다.

장바구니 화면은 담긴 상품 목록과 총액을 보여주고 주문 페이지로 이동한다. 같은 상품은 중복 추가하지 않는다.

주문 페이지는 주문한 e-book 내역, 전체 금액, 입금계좌 정보와 간단한 설명을 보여준다. 입력 항목은 이름, 연락처, 현금영수증 또는 세금계산서 선택, e-mail이다.

관리자 페이지는 로그인 후 접근한다. 관리자는 상품 등록/수정/삭제를 할 수 있고, 주문 목록에서 입금확인과 발송완료를 체크할 수 있다.

## 데이터

`products`는 상품명, 저자/소속, 가격, 소개, 목차, YouTube 멤버십 여부, 표지 이미지 URL, 판매 여부를 저장한다.

`orders`는 주문자 이름, 연락처, 이메일, 증빙 선택, 전체 금액, 입금확인 여부, 발송완료 여부를 저장한다.

`order_items`는 주문과 상품의 연결, 주문 당시 상품명과 가격을 저장한다.

## 운영 범위

1차 구현에서는 실제 이메일 자동 발송은 제외한다. 관리자가 입금 확인 후 e-book을 직접 이메일로 발송하고, 관리자 페이지에서 발송완료 체크를 하면 목록에 발송완료 상태가 표시된다.

## 검증

주문 생성, 장바구니 처리, 상품 등록, 관리자 상태 변경은 자동 테스트로 검증한다. 화면은 로컬 서버에서 직접 확인할 수 있게 한다.
